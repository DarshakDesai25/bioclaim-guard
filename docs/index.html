The problem

AI generates answers.
BioClaim Guard verifies them.

LLMs produce medical claims that sound authoritative — but may be overgeneralized, outdated, or factually wrong.

Silent failures in medical AI

Overgeneralization — applying specific findings to all patients
Causation vs correlation confusion
Animal studies used for human clinical conclusions
Outdated findings contradicted by newer research
Effect size exaggeration — inflating magnitude of risks

Technical design

How it works

Two core AI components — Retrieval-Augmented Generation and structured Prompt Engineering.

1

PICO extraction

Claude decomposes claims into Population, Intervention, Comparison, and Outcome — the gold standard in evidence-based medicine.

2

PubMed RAG

PICO components drive a targeted query to NCBI E-utilities, retrieving top-k peer-reviewed studies from 37M+ citations.

3

Evidence alignment

Claude compares the claim against studies using structured prompts that check for overgeneralization, causation errors, and more.

4

Structured verdict

Outputs a classification, confidence level, evidence quality rating, key issues, and a clinical recommendation.

5

Cited sources

Every verdict links to original PubMed studies — users can trace every conclusion back to primary evidence.

6

Transparent uncertainty

When evidence is absent or weak, the system classifies as "Insufficient Evidence" — never fabricating certainty.

Ethical framework

Responsible AI in healthcare

🔒

No personal data

Only publicly available, de-identified PubMed data. No user or patient data collected or stored.

🔗

Full transparency

Every verdict includes citations linking to original PubMed studies. All reasoning is traceable.

⚖

Bias awareness

The system explicitly flags population mismatches in PICO analysis, acknowledging trial diversity gaps.

📊

Uncertainty first

When evidence is weak, the system classifies as "Insufficient Evidence" rather than fabricating confidence.

🚫

Not medical advice

BioClaim Guard is a research tool. Every response includes a professional consultation disclaimer.

📚

Open source

All code is publicly available on GitHub for inspection, reproducibility, and community improvement.
